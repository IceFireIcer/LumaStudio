# Luma Studio (光影工作室)

> 一款自托管的网页版图片查看器与 Lightroom 风格图片编辑器。

---

## 一、项目是什么

Luma Studio 是一个 **单服务器 Web 应用**，用户在浏览器中完成照片的上传、浏览、编辑、筛选、导出等全部操作，照片以真实文件保存在服务器磁盘上（不是浏览器缓存），重启不丢失。

**技术栈：**
- 后端：Node.js + Express + sharp (libvips 图像处理) + exifr/piexifjs (EXIF 元数据) + yazl (ZIP 打包)
- 前端：原生 HTML + CSS + JavaScript（无框架、无构建步骤）
- 存储：本地 JSON 文件索引 + 磁盘文件

**开发者：** IceFire_Icer

## 二、已实现的功能

### 📷 相册浏览
- 拖拽或点击上传（JPG / PNG / WebP / AVIF / GIF / TIFF / BMP）
- 服务器端生成 WebP 缩略图，网格瀑布布局
- 卡片悬浮动画（上浮 + 图片放大 + 操作按钮弹出）
- 灯箱大图查看，`←` `→` 键切换，`Esc` 关闭
- 单张下载原图（卡片 + 灯箱 + EXIF 页三处入口）
- 重命名（编辑器底部双击文件名 + 灯箱内按钮）
- 删除单张 / 清空全部

### 🎨 编辑器（Lightroom 风格）
- **调整面板**：亮度、对比度、饱和度、色相、锐化、模糊、黑白开关 — CSS 实时预览
- **一键滤镜预设**：原图 / 鲜艳 / 柔和 / 复古 / 黑白 / 高对比
- **变换面板**：左右旋转 90°、水平/垂直翻转、**交互式裁剪**（自由 + 1:1 / 4:3 / 16:9 / 3:4 比例，可拖拽缩放）
- **尺寸面板**：精确像素输入（锁定比例）+ 25% / 50% / 75% / 100% 快速缩放
- **导出面板**：格式选择（JPEG / PNG / WebP / AVIF）+ 质量滑块 + 实时体积预估 + 覆盖原图/另存副本
- **下载到本地**：处理后不落库，直接回传字节供浏览器下载
- **重置全部**：一键恢复所有调整、变换、尺寸

### ⭐ 选片/评分系统
- 1–5 星评分（键盘 `1`–`5`，`0` 清除）
- 精选 / 排除标记（键盘 `P` 精选，`R` 排除）
- 卡片上显示星星徽章和标记状态

### ☑️ 多选 + 批量操作
- 卡片左上角复选框，支持全选 / 取消
- 批量操作栏：标记精选/排除、批量评分、添加到收藏夹、打包 ZIP 下载、批量删除

### 📁 收藏夹（相册管理）
- 创建 / 重命名 / 删除收藏夹
- 将照片添加到收藏夹 / 从收藏夹移除
- 侧栏「收藏夹」导航，浏览收藏夹内容

### 🔍 搜索、筛选、排序
- 顶部工具栏：文件名搜索（带防抖）、星级筛选、标记筛选、格式筛选
- 排序：最新/最早上传、名称正序/倒序、大小、评分

### 🎬 幻灯片
- 全屏黑底自动播放（3 秒间隔）
- `空格` 暂停/继续，`←` `→` 手动切换，`Esc` 退出

### ℹ️ EXIF 元数据
- 查看：相机品牌、型号、镜头、光圈、快门、ISO、焦距、拍摄时间、GPS 坐标等
- 编辑：作者、版权、描述、拍摄时间（仅 JPEG，支持 UTF-8 / 中文）
- 一键抹除全部元数据（隐私保护）
- 下载原图

### ⚙️ 设置页
- 默认导出格式和质量
- 缩略图尺寸
- 主题色（6 个预设 + 自定义色板）
- 存储统计、清空全部

### ℹ️ 关于页
- 版本号、作者、技术栈标签
- 运行时信息：Node 版本、sharp/libvips 版本、照片数量、占用空间、服务器运行时间

### ⌨️ 快捷键
| 键位 | 功能 |
|------|------|
| `1`–`5` | 评分 1–5 星 |
| `0` | 清除评分 |
| `P` | 标记精选 |
| `R` | 标记排除 |
| `←` `→` | 灯箱 / 幻灯片中切换 |
| `空格` | 幻灯片暂停/继续 |
| `Esc` | 关闭灯箱 / 幻灯片 |

## 四、未完成 / 卡住的地方

### 🔴 Electron 桌面版（卡在这一步）

**目标：** 打开应用 = 启动服务器 + 打开窗口；关闭窗口 = 停止服务器 + 退出。

**当前状态：** `electron-main.cjs` 已写好（Express 服务器 + Electron BrowserWindow + 关闭即停），但 **Electron 35 的 `require('electron')` 在 CJS 主进程中返回二进制路径字符串而非模块对象**，导致 `app` / `BrowserWindow` 无法获取。

**已尝试的方案：**
- `require('electron')` → 返回 `"E:\\...\\electron.exe"` (string)
- `require('electron/main')` → `Cannot find module`
- `process._linkedBinding('electron_common')` → `No such binding` + segfault
- 改 `.js` → `.cjs` 解决了 `"type": "module"` 的 ESM 干扰，但 `require('electron')` 仍返回路径

**可能的解决方向（需要另一个 AI 探索）：**
1. **降级 Electron 版本**：尝试 Electron 25 或 26，这些版本的 `require('electron')` 应该正常返回模块对象
2. **ESM 模式**：将 `electron-main.cjs` 改为 `.mjs`，使用 `import { app } from 'electron'`（需确认 Electron 对 ESM 主进程的支持）
3. **使用 `electron/main` 子路径**：某些 Electron 版本通过 `exports` 字段暴露了 `electron/main`，需要检查具体版本的 `node_modules/electron/package.json`
4. **打包方式**：如果无法在开发模式下解决，可能需要在 `electron-builder` 打包后的环境中测试（打包后的二进制行为可能不同）

## 六、⚠️ 特别说明：前端 UI 开发规范

**开发前端页面时，必须严格遵守以下规范：**

1. **不能随意改动现有布局和风格。** 当前 UI 是精心设计的白色主调、毛玻璃侧边栏、圆角卡片、缓动动画系统，所有新增页面/组件必须保持一致。

2. **设计语言固定：**
   - 颜色：`--bg: #ffffff`，`--bg-soft: #f6f7f9`，`--text: #1d1d1f`，`--accent: #0071e3`
   - 圆角：卡片 `18px`，按钮 `980px`（胶囊形），输入框 `12px`
   - 阴影：`0 8px 30px rgba(0,0,0,.07)`（常态），`0 18px 56px rgba(0,0,0,.14)`（悬浮）
   - 动画曲线：`cubic-bezier(.34,1.56,.64,1)`（弹跳），`cubic-bezier(.22,1,.36,1)`（进场）
   - 字体：`-apple-system, BlinkMacSystemFont, "Segoe UI", "PingFang SC", "Microsoft YaHei", sans-serif`

3. **新增功能时：** 先看 `style.css` 中已有的组件样式（`.btn`、`.card`、`.field`、`.slider-row`、`.switch-row`、`.chip`、`.hint`、`.divider` 等），复用它们，不要重新发明。

4. **新增页面时：** 参照「关于页」的结构（`.topbar` + `.settings-card` 容器），保持一致的间距和排版节奏。

5. **新增侧栏导航时：** 复用 `.nav-item` 的 SVG + `<span>` 结构，不要用图标字体或外部图标库。

6. **响应式断点已定义：** `900px`（编辑器/EXIF 单列）和 `680px`（侧栏折叠为图标），新增内容需适配这两个断点。

## 七、如何运行

### bat 脚本版（推荐，开箱即用）
```
进入 LumaStudio-bat/
双击 install.bat → 自动安装到系统 + 创建桌面快捷方式
以后双击桌面「Luma Studio」图标即可启动
```

### 源码版（开发/调试）
```
进入 LumaStudio/
npm install
node server.js
浏览器打开 http://localhost:3000
```

### Electron 版（未完成）
```
进入 LumaStudio-electron/
npm install
npm run electron   // 会报错，见上方说明
```
